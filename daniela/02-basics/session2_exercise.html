7.1 Exercise: Python Basics with Real(-ish) Data

We have a small reaction time dataset: 20 participants, each completing trials across two stimulus types (word, image) and two conditions (congruent, incongruent).

Work through the cells below. Each cell tells you what to do — write your code underneath the comment.

# 1a. Import the csv module (built-in, no install needed)
# 1b. Open 'data/reaction_times.csv' using csv.DictReader
#     and load all rows into a list called 'trials'
#     Print how many rows you loaded.
# 1c. Print the first row to see what it looks like.
#     What type is it? (Hint: use type())

7.3 Part 2 — Exploring with Variables & Types

# 2a. Grab the first trial and print its reaction time (the 'rt' column).
#     Check the type — is it a float or a string? Why?
# 2b. Convert the 'rt' value to a float and the 'age' value to an int.
#     Print both with their types to confirm.
# 2c. How many unique participants are in the dataset?
#     Hint: build a list of all participant_id values,
#     then wrap it in set() to get unique values, then len().
# 2d. What stimulus types and conditions are in the data?
#     Print the unique values for both columns.

7.4 Part 3 — Lists & Filtering

# 3a. Build a list of all rt values (as floats) across all trials.
#     Print the first 5 values.
# 3b. Using only built-in functions (min, max, sum, len),
#     compute and print the min, max, and mean RT.
# 3c. Use a list comprehension to keep only the RT values
#     from 'incongruent' trials.
#     How many trials is that? What is the mean RT?
# 3d. Now do the same for 'congruent' trials.
#     Print the mean RT for both conditions side by side.
#     Is the incongruent condition slower, as expected?

7.5 Part 4 — Loops & Conditionals

# 4a. Loop over all trials and count how many were correct
#     and how many were incorrect.
#     Note: 'correct' is stored as the string 'True'/'False' — handle that!
# 4b. Print each participant's ID and their number of trials,
#     using a dict to count as you loop.
#     Expected: each participant should have 20 trials.
# 4c. Flag 'outlier' trials: RT < 0.15 s or RT > 1.0 s.
#     Print the trial_id and RT of any outlier trials you find.
#     How many are there?

7.6 Part 5 — Functions

# 5a. Write a function called mean_rt(trials, condition=None)
#     that takes the full list of trials and returns the mean RT.
#     If condition is given (e.g. 'congruent'), filter to that condition first.
#     Test it with and without the condition argument.
# 5b. Write a function called participant_summary(trials, pid)
#     that takes the trial list and a participant ID string,
#     and returns a dict with keys: 'n_trials', 'mean_rt', 'accuracy'
#     (accuracy = proportion correct, as a float between 0 and 1).
#
#     Test it on participant 'P01'.
# 5c. Use your function to loop over all participant IDs
#     and print a summary line for each one, e.g.:
#     P01 | mean RT: 0.442 s | accuracy: 97.0%
