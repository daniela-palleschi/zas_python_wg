6 Learning Objectives

  • Write basic Python: variables, if/else, for loops
  • Understand lists, tuples, and dicts
  • Write and call a simple function
  • Read a CSV file and print its contents

7 1 Variables & Data Types

Python is dynamically typed — you never declare a type, Python infers it. Use type() to inspect at any time.

rt        = 0.423          # float  (reaction time in seconds)
n_trials  = 48             # int
condition = 'congruent'    # str
correct   = True           # bool

print(type(rt))            # <class 'float'>
print(type(n_trials))      # <class 'int'>
R / Matlab 
R:      rt <- 0.423   │   class(rt) => 'numeric'
Matlab: rt = 0.423;   │   class(rt) => 'double'

Python has no implicit numeric promotion — int stays int unless you divide (5 / 2 = 2.5).
Integer division: 5 // 2 = 2 │ Remainder: 5 % 2 = 1

7.1 Common type conversions

int('42')          # => 42
float('3.14')      # => 3.14
str(99)            # => '99'
bool(0)            # => False  (0, '', None, [], {} are all falsy)
Watch out

Strings are immutable in Python — you cannot do s[0] = 'X'. Build a new string instead.

8 2 Data Structures: Lists, Tuples, Dicts

8.1 Lists [ ]

Ordered, mutable, can hold mixed types. The everyday workhorse.

rts = [0.423, 0.381, 0.512, 0.299, 0.478]

rts[0]          # 0.423  — first element (0-indexed!)
rts[-1]         # 0.478  — last element
rts[1:3]        # [0.381, 0.512]  — slice: index 1 up to (not including) 3

rts.append(0.350)          # add one item at the end
rts.extend([0.410, 0.390]) # add multiple items
len(rts)                   # number of items
sorted(rts)                # returns a new sorted list (does not modify rts)
rts.sort()                 # sorts rts in place
R / Matlab 
R:      rts <- c(0.423, 0.381, ...)  │  rts[1]  (1-indexed!)
Python: rts = [0.423, 0.381, ...]   │  rts[0]  (0-indexed)

R’s length() → Python’s len()
R’s append() → Python’s .append() or .extend() for multiple items

8.2 Tuples ( )

Like lists but immutable — contents cannot change after creation. Good for things that should stay fixed (coordinates, column headers, return values from a function).

participant = ('P07', 'Klara', 24)   # id, name, age
participant[0]    # 'P07'
# participant[0] = 'P99'  => TypeError! tuples are immutable

# Unpack a tuple into variables:
pid, name, age = participant
print(name)       # Klara

8.3 Dictionaries { }

Key–value pairs. Think of named columns, or R’s named lists. Keys must be unique.

trial = {
    'participant': 'P07',
    'condition':   'congruent',
    'rt':          0.423,
    'correct':     True
}

trial['rt']              # 0.423  — access by key
trial['rt'] = 0.399      # update a value
trial['block'] = 1       # add a new key

trial.keys()             # dict_keys(['participant', 'condition', ...])
trial.values()           # dict_values(['P07', 'congruent', ...])
trial.get('block', 0)    # safe access: returns 0 if key missing
R / Matlab 
R named list:  trial <- list(participant='P07', rt=0.423)
               trial$rt  or  trial[['rt']]
Python dict:   trial = {'participant': 'P07', 'rt': 0.423}
               trial['rt']
Tip: When to use which?
  • List = ordered sequence of similar things
  • Tuple = fixed record that should not change
  • Dict = labelled data where you look things up by name

9 3 Conditionals

Python uses indentation to define blocks — there are no curly braces or end keywords. Consistent indentation is not optional; it is the syntax.

rt = 0.423

if rt < 0.3:
    print('Fast response')
elif rt < 0.5:
    print('Normal response')
else:
    print('Slow response')
R vs Python
R Python
if (rt < 0.3) { if rt < 0.3:
print('Fast') print('Fast')
} else if (rt < 0.5) { elif rt < 0.5:
print('Normal') print('Normal')
} else { else:
print('Slow') print('Slow')
}

Combining conditions:

if rt < 0.5 and correct == True:
    print('Valid fast trial')

if rt > 1.0 or correct == False:
    print('Exclude this trial')

if condition != 'practice':   # != means 'not equal'
    print('Experimental trial')
Watch out

Python uses and, or, not — not &&, ||, ! like R.
Equality check is ==, not = (which is assignment).

10 4 Loops

10.1 for loop — iterating over a list

rts = [0.423, 0.381, 0.512, 0.299, 0.478]

for rt in rts:
    print(round(rt, 3))

10.2 range() — counting loops

for i in range(5):        # 0, 1, 2, 3, 4
    print(i)

for i in range(1, 6):     # 1, 2, 3, 4, 5
    print(i)
R / Matlab 
R:      for (i in 1:5) { ... }
Python: for i in range(1, 6): ...

Note: range(1, 6) gives 1 to 5 — the stop value is excluded.

10.3 enumerate() — loop with index

conditions = ['congruent', 'incongruent', 'neutral']

for i, cond in enumerate(conditions):
    print(i, cond)
# 0 congruent
# 1 incongruent
# 2 neutral

10.4 List comprehensions — compact loops

A Pythonic way to build a new list from an existing one in a single line.

rts_ms = [rt * 1000 for rt in rts]           # convert to milliseconds
fast    = [rt for rt in rts if rt < 0.4]     # filter: keep only fast RTs
R / Matlab 
R:      rts_ms <- rts * 1000          (vectorised — no loop needed)
Python: rts_ms = [rt * 1000 for rt in rts]

With numpy/pandas, Python is also vectorised. List comprehensions are for plain lists.

11 5 Functions

Define once, reuse anywhere. Functions help you avoid copy-pasting the same code block.

def mean_rt(times):
    """Return the mean of a list of reaction times."""
    return sum(times) / len(times)

# Call it:
avg = mean_rt([0.423, 0.381, 0.512])
print(f'Mean RT: {avg:.3f} s')   # Mean RT: 0.439 s

11.1 Default arguments

def summarise_rt(times, unit='s'):
    avg = sum(times) / len(times)
    if unit == 'ms':
        avg = avg * 1000
    return avg

summarise_rt(rts)            # uses default: seconds
summarise_rt(rts, unit='ms') # milliseconds

11.2 Returning multiple values

def rt_stats(times):
    return min(times), max(times), sum(times) / len(times)

lo, hi, avg = rt_stats(rts)   # unpack the returned tuple
print(f'Min: {lo:.3f}  Max: {hi:.3f}  Mean: {avg:.3f}')
R / Matlab 
R:      my_fun <- function(x, unit = 's') { ... }
Python: def my_fun(x, unit='s'):  ...

R returns the last expression; Python needs an explicit return.
R returns a list for multiple values; Python returns a tuple (unpack with a, b = fun()).

Tip

Write a docstring (the triple-quoted string right after def) — it shows up when you call help(my_fun) or hover over it in VS Code.

12 6 Reading a CSV File

Two approaches: the built-in csv module (no dependencies), or pandas (more powerful, covered in Session 3).

12.1 Option A — built-in csv module

import csv

with open('data/scores.csv', newline='') as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row['name'], row['score'])

csv.DictReader reads each row as a dict with column names as keys — no extra libraries needed.

12.2 Option B — pandas (preview of Session 3)

import pandas as pd

df = pd.read_csv('data/scores.csv')
print(df.head())           # first 5 rows
print(df['score'].mean())  # column mean
R / Matlab 
R:               df <- read.csv('data/scores.csv')
Python (pandas): df = pd.read_csv('data/scores.csv')

Both give you a table-like object. Pandas’ DataFrame ≈ R’s data.frame.

Tip

Always use with open(...) — it automatically closes the file even if an error occurs.
